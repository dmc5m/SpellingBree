## Architecture

SpellingBee is a **kids' spelling game** for the UFLI phonics curriculum, built as a Next.js 15 app (React 19, Tailwind CSS v4, Framer Motion). It's a PWA targeting iOS/Safari with careful audio handling.

### File structure

- `lib/config.ts` — Timeouts and constants (HINT_TIMEOUT_MS, TTS_RATE, CORRECT_TO_LEVEL_UP)
- `lib/words.ts` — Weekly UFLI word lists by level. **Update this file when David brings new words, then regenerate audio.**
- `hooks/use-audio.ts` — Audio engine: static MP3 playback, hint buffer chain, AudioContext management, iOS unlock
- `hooks/use-spelling-game.ts` — Game state: phase machine, level progression, localStorage persistence, answer checking, forced-typing sub-state
- `components/game-board.tsx` — Main game UI (stats bar, input, buttons, feedback, show-word mode)
- `components/splash-screen.tsx` — Loading screen
- `components/audio-unlock.tsx` — iOS/Safari audio enable screen
- `components/celebration.tsx` — Completion screen after all levels
- `components/confetti.tsx` — Canvas-based confetti with requestAnimationFrame
- `app/page.tsx` — Thin shell that wires hooks together and renders screens by GamePhase
- `app/api/hint/route.ts` — Vercel serverless API: calls Azure OpenAI for teacher feedback + Azure Speech REST API for TTS, returns MP3
- `scripts/generate-audio.mjs` — One-time script to generate all static MP3s via Azure Speech REST API
- `public/audio/` — Pre-generated Bree Dragon HD MP3s (words, phrases, buffer chain lines)

### Game phases

`loading` → `audio-unlock` (iOS) or `splash` → `playing` → `completed`. Error state available at `splash` → `error` with retry.

### Audio architecture

**No Flask dependency.** All audio is either pre-generated static files or generated by a Vercel API route.

- **Word pronunciation:** Static MP3s in `public/audio/words/{word}.mp3`
- **Game phrases:** Static MP3s in `public/audio/phrases/` (level announcements, correct, level-up, completed, now-you-got-it)
- **Hint buffer chain:** Static MP3s in `public/audio/buffer/` (buffer-1 through buffer-4b)
- **GPT hints:** `POST /api/hint` calls Azure OpenAI + Azure Speech REST API, returns MP3

### Hint buffer chain

When a child misspells a word, pre-recorded Bree audio plays sequentially while GPT generates a personalized hint in the background:

1. "Hmmm, I'll think of a good hint for you." → wait 1s → if hint ready, play it
2. "I'm working on a good idea for how to explain this." → wait 1s → if ready, play 4a then hint
3. "Hmm, this is a tricky one even for me!" → wait 1s → if ready, play 4a then hint
4a. "Okay, here's what I'm thinking." (transition into hint, only after 2 or 3)
4b. "You know what, let me just show you this one. We'll get it next time!" (give up → forced typing)

Buffer lines 1-3 at slow rate (-30%), 4a/4b at normal rate (0%).

### Forced-typing sub-state

After buffer 4b (hint never arrived), the correct word is displayed on screen and the child must type it correctly before advancing. Uses `lastResult.kind === "show-word"` in game state.

### Audio design

- Single shared AudioContext, reused across the session
- `playStatic(path)` loads and plays a single MP3; `playSequence(paths)` chains multiple
- Decoded AudioBuffers cached in a Map (no eviction needed, word lists are small)
- `stop()` cancels the hint chain, aborts in-flight requests, and stops playing audio
- `isInHintChain` locks the entire UI (buttons + input) during the buffer chain including 1s pauses
- Safety timeout on playback in case Safari's `onended` never fires
- `visibilitychange` handler resumes AudioContext after iOS backgrounding
- Buffer chain MP3s preloaded on mount

### When David brings new words

1. Update `lib/words.ts` with the new word list
2. Update `scripts/generate-audio.mjs` WORDS object to match
3. Run: `source ../gratitude-flask/.env && node scripts/generate-audio.mjs`
4. Commit the updated word list and new MP3 files

### Environment variables (Vercel)

For the `/api/hint` route:
- `AZURE_OPENAI_ENDPOINT` — Azure OpenAI endpoint URL
- `AZURE_OPENAI_KEY` — Azure OpenAI API key
- `AZURE_OPENAI_DEPLOYMENT` — Model deployment name
- `AZURE_OPENAI_API_VERSION` — API version (defaults to 2024-12-01-preview)
- `AZURE_SPEECH_KEY` — Azure Speech Service subscription key
- `AZURE_SPEECH_REGION` — Azure Speech Service region
